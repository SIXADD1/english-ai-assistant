# 英语 AI 助手 - 技术架构文档

## 1. 整体架构

```
┌──────────────────────────────────────────────────────┐
│                      浏览器                           │
└──────────────┬───────────────────────────────────────┘
               │ HTTPS
               ▼
┌──────────────────────────────────────────────────────┐
│   Nginx（宝塔面板管理）                                │
│   ┌──────────────┐   ┌──────────────────────────┐    │
│   │ 静态资源托管  │   │ 反向代理 /api/* → 后端   │    │
│   │ dist/        │   │                          │    │
│   └──────────────┘   └───────────┬──────────────┘    │
└──────────────────────────────────┼───────────────────┘
                                   │
              ┌────────────────────┼────────────────────┐
              ▼                    ▼                    ▼
┌─────────────────────┐ ┌──────────────────┐ ┌──────────────────┐
│  前端（dist/）       │ │  后端（api/）     │ │  AI 服务          │
│  纯静态文件          │ │  Node.js 3001    │ │  Python 8000     │
│  Nginx 直接托管      │ │  Express         │ │  FastAPI          │
└─────────────────────┘ └────────┬─────────┘ └────────┬─────────┘
                                 │                     │
                                 ▼                     ▼
                        ┌──────────────────┐  ┌──────────────────┐
                        │  PostgreSQL       │  │  DeepSeek API     │
                        │  （唯一数据库）     │  │  （AI 批改引擎）   │
                        └──────────────────┘  └──────────────────┘
```

**三个独立服务：**

| 服务 | 语言 | 端口 | 部署方式 |
|------|------|------|---------|
| 前端 | TypeScript → 编译为 JS | Nginx 80 | `npm run build` → `dist/` 静态文件 |
| 后端 | JavaScript (Node.js) | 3001 | 源码直接跑，`node src/app.js` |
| AI 服务 | Python | 8000 | `python3 main.py` |

- `vite.config.ts` 开发代理：前端 `/api/*` 请求自动转发到后端 3001 端口

## 4. AI 服务架构

### 4.1 调用链路

```
用户提交作文
    │
    ▼
POST /api/writing/submit  （后端 app.js）
    │ 保存草稿到 PostgreSQL
    │ fetch() → AI 服务
    ▼
POST /api/correct  （Python FastAPI 8000 端口）
    │ 组装 Prompt + 作文
    │ httpx → DeepSeek API
    ▼
DeepSeek API
    │ 返回 JSON 批改结果
    ▼
后端解析结果
    │ 存入 corrections 表
    ▼
返回给前端展示
```

### 4.2 技术栈

| 技术 | 用途 |
|------|------|
| FastAPI | Python Web 框架 |
| httpx | 异步 HTTP 客户端，调 DeepSeek |
| Pydantic | 数据校验 |
| DeepSeek API | 实际 AI 引擎（替代 GPT-4/Claude） |

### 4.3 批改流程

1. 接收作文内容 + 题目信息 + 用户等级（四级/六级）
2. 根据等级选择对应的评分 Prompt（四级偏重格式规范性，六级偏重论证深度）
3. 调用 DeepSeek `chat/completions` 接口
4. 解析返回的 JSON：分数、扣分明细、错误列表、优化建议、改写版本
5. 返回给后端，存入数据库

### 5.2 UUID 生成

所有主键使用 `md5(random()::text || clock_timestamp()::text)::uuid`，不依赖 `uuid-ossp` 扩展（兼容所有 PostgreSQL 环境）。

## 6. 部署架构

### 6.1 部署文件

只需 4 个目录：

| 目录 | 作用 | 部署方式 |
|------|------|---------|
| `dist/` | 前端（编译产物，3 个文件） | Nginx 静态托管 |
| `api/` | 后端源码 | 宝塔 Node 项目管理器 → `node src/app.js` |
| `ai-service/` | AI 服务源码 | 终端 `python3 main.py` |
| `migrations/` | 数据库脚本 | PostgreSQL 导入 |

### 6.2 部署流程

```
本地                          服务器
─────────────────────────────────────────
.\pack-deploy.ps1             → deploy-package.zip 上传
  ├── npm run build             → 解压到 /www/wwwroot/fad.asia/
  ├── 压缩 4 个目录              → 配置 .env
  └── 生成 deploy-package.zip   → 导入 SQL
                                → 启动 3 个服务
                                → Nginx 配置完成
```

详细部署步骤见 [DEPLOY.md](../../DEPLOY.md)。

## 7. 安全设计

| 措施 | 实现 |
|------|------|
| 密码加密 | bcrypt 哈希存储，不存明文 |
| JWT 认证 | 登录后签发 token，所有 API 请求携带 |
| 权限分级 | `authenticateToken`（普通用户）/ `authenticateAdmin`（管理员） |
| 安全头 | helmet 中间件自动添加 |
| 跨域控制 | CORS 配置 `ALLOWED_ORIGINS` 白名单 |
| SQL 注入防护 | 全部使用参数化查询（`$1, $2` 占位符） |
| AI Key 保护 | 存在数据库 `system_configs` 表，不暴露给前端 |
| 频率限制 | `express-rate-limit`；登录 5次/分钟，注册 3次/小时，超限提示等待时间 |
